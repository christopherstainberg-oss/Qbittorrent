Rules are evaluated top-to-bottom; the first match wins. Each condition is optional — a rule matches when all the conditions you set pass. Comma-separate list fields. Changes are saved to config.yaml.

Trigger → action automations: when a torrent matches the conditions, run the chosen action. Conditions work like Rules (a torrent matches when all the ones you set pass). These run automatically in the pipeline (poll / completion webhook) and on demand below. “Only completed torrents” limits an automation to fully-downloaded ones.

Library relocation (Sonarr/Radarr-style)

Move completed torrents in a category out to a library path that qBittorrent can't reach — the app hardlinks (or copies) the files itself. Requires mounting your download folder and each destination into this container. Hardlink keeps seeding and uses no extra space when source and destination share a filesystem.
